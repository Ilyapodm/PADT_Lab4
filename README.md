## Ordinal

`Ordinal` хранит ограниченные ординалы вида:

```text
ω * k + n
```

где:

* `k` — количество бесконечных блоков;
* `n` — конечная часть.



### Сложение

Сложение ординалов некоммутативно:

```text
5 + ω = ω
ω + 5 != 5 + ω
ω + 5 + ω + 3 = ω * 2 + 3
```

Если справа добавляется бесконечная часть, конечный хвост слева поглощается.

### subtract_prefix

`subtract_prefix(prefix)` — это не обычное арифметическое вычитание, а вычисление локального индекса после префикса.

Он ищет такой `x`, что:

```text
prefix + x == current
```

Используется, например, в `ConcatNode`, чтобы перевести глобальный индекс во второй последовательности в локальный индекс.

Примеры:

```text
(7).subtract_prefix(3) = 4

(ω + 5).subtract_prefix(ω) = 5

(ω + 7).subtract_prefix(ω + 5) = 2

(ω * 2 + 4).subtract_prefix(ω + 7) = ω + 4
```

Если `prefix` больше текущего ординала, выбрасывается исключение.

## Генераторы

### Generator

`Generator<T>` — базовая абстракция для ленивого порождения элементов.

* `get_next()` — последовательное чтение элементов;

`clone()` нужен для копирования генератора вместе с его текущим состоянием чтения.

### EmptyGenerator

`EmptyGenerator<T>` представляет пустую последовательность длины `0`.

Свойства:

* `has_next()` всегда возвращает `false`;
* `get_next()` выбрасывает исключение;
* `length()` возвращает `0`;

Используется как нулевой объект для пустой `LazySequence`.

### SingletonGenerator

`SingletonGenerator<T>` представляет ленивую последовательность из одного элемента.

Свойства:

* `length()` возвращает `1`;
* `get_next()` один раз возвращает сохранённое значение, затем выбрасывает исключение;
* `clone()` копирует генератор вместе с состоянием: был ли элемент уже прочитан.


### SequenceGenerator

`SequenceGenerator<T>` адаптирует обычный `Sequence<T>` под интерфейс ленивого генератора.

Делает копию последовательности. Не работает с LazySequence!

Свойства:

* `get_next()` читает элементы по порядку и сдвигает внутренний индекс;
* `length()` равна размеру исходной последовательности;
* `clone()` копирует генератор вместе с текущим индексом чтения.

### FunctionGenerator

`FunctionGenerator<T>` создаёт ленивую последовательность по правилу вида `a_i = f(i)`.

Особенности:

* хранит функцию, длину и текущий индекс чтения;
* `get_next()` вычисляет следующий элемент и сдвигает индекс;
* поддерживает конечную длину или `ω`;
* не хранит вычисленные значения, так как элемент получается напрямую по индексу.

Примеры последовательностей: натуральные числа, квадраты, арифметические прогрессии.


### RecurrenceGenerator

`RecurrenceGenerator<T>` создаёт ленивую последовательность по рекуррентному правилу.

Он хранит:

- функцию перехода;
- кольцевое окно последних `k` элементов и само `k`;
- текущий индекс чтения;
- длину последовательности.

`get_next()` сначала возвращает начальные значения, затем вычисляет следующий элемент по окну и сдвигает окно через `RingBuffer`.

Генератор не хранит всю историю значений и не выполняет полную мемоизацию.  


### RingBuffer

`RingBuffer<T>` — вспомогательный фиксированный кольцевой буфер для `RecurrenceGenerator`.

Он хранит окно последних `k` значений без физического сдвига элементов. При добавлении нового значения самый старый элемент перезаписывается, а индекс начала окна сдвигается по кругу.

Используется для эффективного обновления рекуррентного окна:

* `get(0)` возвращает самый старый элемент окна;
* `get(size - 1)` возвращает самый новый элемент;
* обновление окна выполняется за `O(1)`;
* память фиксирована и равна `O(k)`;
* полный кэш ранее вычисленных элементов не хранится.

## Ноды


### Node

`Node<T>` — абстрактный узел ленивой последовательности.

Он не является генератором и не выполняет последовательное чтение.  
`Node` отвечает за доступ к значению по ординальному индексу и за маршрутизацию индексов внутри операций.

Основная идея:

- `LazySequence` хранит корневой `Node`;
- операции `concat`, `map`, `subsequence`, `insert` создают новые узлы;
- каждый узел знает свою `Ordinal`-длину;
- `value_at(index)` возвращает значение по индексу;
- `clone()` создаёт независимую копию узла вместе с его кэшем.

### SourceNode

`SourceNode<T>` — базовый узел-источник для `LazySequence`.

Он оборачивает `Generator<T>` и хранит собственный плотный кэш уже вычисленных значений.

Поведение:

- хранит `Generator<T>*`, потому что генераторы полиморфны;
- хранит `MutableArraySequence<T>` как кэш;
- кэш является плотным префиксом последовательности;
- если запрошенный элемент уже есть в кэше, он возвращается сразу;
- если элемента нет, `SourceNode` доращивает кэш через `generator_->get_next()`;
- `clone()` копирует генератор в текущем состоянии и копирует кэш;


### ConcatNode

`ConcatNode<T>` — операционный узел ленивой последовательности, представляющий конкатенацию двух `Node<T>`.

Поведение:

- хранит независимые копии левого и правого узлов через `clone()`;
- сохраняет уже накопленные кэши дочерних узлов;
- длина равна `left.length() + right.length()`;
- `value_at(index)` маршрутизирует индекс:
  - если `index < left.length()`, запрос идёт в левый узел;
  - иначе вычисляется локальный индекс через `subtract_prefix(left.length())`, и запрос идёт в правый узел;
- не имеет собственного кэша, потому что использует кэши дочерних узлов;
- поддерживает случаи `finite + finite`, `finite + ω`, `ω + finite`, `ω + ω`.

### SubsequenceNode

`SubsequenceNode<T>` — операционный узел, представляющий подпоследовательность исходного `Node<T>` в диапазоне `[start, end_exclusive)`.

Узел не материализует элементы и не хранит собственный кэш. Он владеет клоном исходного узла и только переводит локальный индекс подпоследовательности в индекс источника:

```text
sub[i] = source[start + i]
```

Кэширование выполняется внутри клонированного источника. Благодаря этому `SubsequenceNode` остаётся ленивым, независимым от исходной последовательности и сохраняет уже материализованные значения через `clone()`.

### SetNode

`SetNode<T>` — операционный узел, представляющий ленивую замену одного элемента исходного `Node<T>`.

Узел хранит индекс замены, новое значение и независимый клон источника. Длина последовательности не меняется.

```text
set[i] = value,     если i == index
set[i] = source[i], иначе
```

`SetNode` не изменяет исходную последовательность и не влияет на правило генерации последующих элементов.

### InsertAtNode

`InsertAtNode<T>` — операционный узел, представляющий ленивую вставку одного значения в исходный `Node<T>` по заданному `Ordinal`-индексу.

Узел не материализует всю последовательность. Он хранит клонированный источник, индекс вставки и вставляемое значение. При доступе к элементу `InsertAtNode` маршрутизирует индекс:

```text
i < insert_index   -> source[i]
i == insert_index  -> inserted_value
i > insert_index   -> source[shifted_i]
```

Для конечных участков после вставки индекс сдвигается назад на `1`. Для переходов через `ω`-границы сдвиг не применяется, чтобы не ломать ординальную структуру последовательности.

`InsertAtNode` владеет независимым клоном источника, поэтому сохраняет ленивость, кэширование и независимость от исходного узла.

### RemoveAtNode

`RemoveAtNode<T>` — операционный узел, представляющий ленивое удаление одного элемента из исходного `Node<T>` по `Ordinal`-индексу.

Узел владеет клоном источника, хранит удаляемый индекс и новую длину. Элементы до удалённого индекса возвращаются без изменений, элементы после него в том же ординальном блоке берутся со сдвигом на `+1`.

```text
result[i] = source[i],     если i < removed_index
result[i] = source[i + 1], если i >= removed_index в том же блоке
```

Для удаления конечного элемента из бесконечного блока длина не меняется: `ω -> ω`. Для удаления элемента из конечного хвоста длина уменьшается на `1`.

### AppendNode

`AppendNode<T>` — операционный узел, добавляющий один элемент в конец ленивой последовательности.

Узел владеет клоном исходного `Node<T>`, хранит добавляемое значение и индекс вставленного элемента:

```text
append_index = source.length()
```

При обращении по индексу:

```text
if index == append_index:
    return appended_value
else:
    return source[index]
```

Для конечной последовательности `append` работает как обычное добавление в конец. Для бесконечной последовательности добавленный элемент находится на трансфинитном индексе `ω`, поэтому длина становится `ω + 1`.

`AppendNode` не имеет собственного кэша для элементов источника: кэширование выполняется внутри клонированного `source`.

### PrependNode

`PrependNode<T>` — операционный узел, добавляющий один элемент в начало ленивой последовательности.

Узел хранит добавленное значение и независимый клон исходного `Node<T>`. При обращении к индексу `0` возвращается добавленное значение, а остальные конечные индексы перенаправляются в источник со сдвигом:

```text
prepended[0] = value
prepended[i] = source[i - 1], если i конечный и i > 0
```

Для трансфинитных индексов сдвиг не применяется: `prepended[ω + n] = source[ω + n]`. Это сохраняет корректную ординальную структуру последовательности.

## LazySequence

`LazySequence<T>` — фасад над деревом ленивых `Node<T>`, представляющий конечную или трансфинитную последовательность.

`LazySequence` не является потоком и не выполняет последовательное чтение. У него нет `get_next()`: доступ к элементам происходит только по индексу через корневой узел.

Основная схема:

```text
Generator<T> -> SourceNode<T> -> operation Node<T> -> LazySequence<T>
```

`LazySequence<T>` хранит:

```text
Node<T>* root_node_
```

и делегирует ему:

```text
get(index)              -> root_node_->value_at(Ordinal(index))
get(ordinal_index)      -> root_node_->value_at(ordinal_index)
get_ordinal_length()    -> root_node_->length()
get_materialized_count()-> root_node_->materialized_count()
```

Поддерживаемые источники:

* пустая последовательность;
* массив `T*`;
* обычный `Sequence<T>`;
* `FunctionGenerator<T>`;
* `RecurrenceGenerator<T>` через `RingBuffer<T>`;
* `RecurrenceGenerator<T>` через массив последних значений.

Операции `append`, `prepend`, `set`, `insert_at`, `remove_at`, `concat`, `subsequence` не изменяют исходную последовательность. Вместо этого они создают новую `LazySequence<T>` с новым корневым операционным узлом:

```text
append      -> AppendNode
prepend     -> PrependNode
set         -> SetNode
insert_at   -> InsertAtNode
remove_at   -> RemoveAtNode
concat      -> ConcatNode
subsequence -> SubsequenceNode
```

Каждая операция сохраняет ленивость: результат не материализуется целиком, а индексный запрос маршрутизируется внутри дерева узлов.

Копирование `LazySequence` выполняется через `Node::clone()`, поэтому производные последовательности независимы от исходных и сохраняют уже накопленный кэш.

Для конечных последовательностей доступны обычные `int`-индексы. Для трансфинитных случаев используется `Ordinal`, например:

```text
0
5
ω
ω + 3
ω * 2 + 1
```

## Streams

В проекте реализованы потоки ввода и вывода:

```text
InputStream<T>
OutputStream<T>
SequenceInputStream<T>
SequenceOutputStream<T>
```

`InputStream<T>` — абстракция последовательного чтения данных. Поток хранит текущую позицию чтения: `input()` возвращает следующий элемент и сдвигает позицию вперёд. Перед использованием поток нужно открыть через `open()`. Метод `reset()` сбрасывает открытый поток в начало, а `close()` закрывает поток.

`SequenceInputStream<T>` работает поверх `Sequence<T>` через общий интерфейс `Enumerable<T>` и `IEnumerator<T>`. Поэтому один и тот же поток может читать данные из `ArraySequence`, `ListSequence`, mutable/immutable последовательностей и `LazySequence` (чтение только начального префикса, не предназначен для перехода через ординальные границы). Источник хранится как `const Sequence<T>*`: поток логически только читает последовательность, а внутренняя мемоизация `LazySequence` не нарушает константность.

`OutputStream<T>` — абстракция последовательной записи данных. Метод `output(value)` добавляет элемент в конец целевой последовательности и возвращает новую позицию потока. Перед записью поток нужно открыть через `open()`.

`SequenceOutputStream<T>` работает с внутренним указателем на текущую `Sequence<T>`. Для mutable-последовательностей `append` изменяет исходный контейнер и возвращает тот же объект. Для immutable-последовательностей `append` создаёт новую версию последовательности, и поток начинает работать с этой новой версией.

`SequenceOutputStream<T>` не владеет исходной последовательностью, переданной в конструктор. Если в процессе записи immutable-последовательность создаёт новые версии, поток владеет актуальной новой версией и удаляет промежуточные версии. Чтобы забрать итоговую последовательность наружу, используется `release_sequence()`. После вызова `release_sequence()` поток отключается и больше не должен использоваться для записи.
