## Ordinal

`Ordinal` хранит ограниченные ординалы вида:

```text
ω * k + n
```

где:

* `k` — количество бесконечных блоков;
* `n` — конечная часть.



### Сложение

Сложение ординалов некоммутативно:

```text
5 + ω = ω
ω + 5 != 5 + ω
ω + 5 + ω + 3 = ω * 2 + 3
```

Если справа добавляется бесконечная часть, конечный хвост слева поглощается.

### subtract_prefix

`subtract_prefix(prefix)` — это не обычное арифметическое вычитание, а вычисление локального индекса после префикса.

Он ищет такой `x`, что:

```text
prefix + x == current
```

Используется, например, в `ConcatGenerator`, чтобы перевести глобальный индекс во второй последовательности в локальный индекс.

Примеры:

```text
(7).subtract_prefix(3) = 4

(ω + 5).subtract_prefix(ω) = 5

(ω + 7).subtract_prefix(ω + 5) = 2

(ω * 2 + 4).subtract_prefix(ω + 7) = ω + 4
```

Если `prefix` больше текущего ординала, выбрасывается исключение.

## Generator

`Generator<T>` — базовая абстракция для ленивого порождения элементов.

Он поддерживает два режима работы:

* `get_next()` — последовательное чтение элементов;
* `get_at(Ordinal index)` — доступ по ординальному индексу, если генератор это поддерживает.

Общая проверка `get_at()` находится в базовом классе:

1. проверяется, поддерживает ли генератор индексный доступ через `supports_get_at()`;
2. проверяется, что индекс меньше длины;
3. вызывается `get_at_impl()` конкретного генератора.

Потомки переопределяют только `get_at_impl()`, чтобы не дублировать проверки.

`clone()` нужен для копирования генератора вместе с его текущим состоянием чтения.

## EmptyGenerator

`EmptyGenerator<T>` представляет пустую последовательность длины `0`.

Свойства:

* `has_next()` всегда возвращает `false`;
* `get_next()` выбрасывает исключение;
* `length()` возвращает `0`;
* `supports_get_at()` возвращает `true`, но допустимых индексов нет.

Используется как нулевой объект для пустой `LazySequence`.

## SequenceGenerator

`SequenceGenerator<T>` адаптирует обычный `Sequence<T>` под интерфейс ленивого генератора.

Он не владеет исходной последовательностью, а хранит указатель на неё.

Свойства:

* `get_next()` читает элементы по порядку и сдвигает внутренний индекс;
* `get_at(index)` получает элемент напрямую из `Sequence<T>` и не меняет текущую позицию чтения;
* `length()` равна размеру исходной последовательности;
* `clone()` копирует генератор вместе с текущим индексом чтения.
